# Принципи програмування проєкту KPZ

У ході розробки клієнтської частини додатка (файл `Client/Form1.cs`) було дотримано наступних принципів:

### 1. SoC (Separation of Concerns — Розподіл відповідальності)

Логіка інтерфейсу користувача (UI) відокремлена від мережевої логіки. Клас `Form1` не знає, як працюють сокети або WCF — він лише викликає методи об'єкта `_connector` та реагує на його події.

- **Доказ у коді:** [Client/Form1.cs рядок 10](https://github.com/ipzk251gor-bot/KPZ/blob/228d7a1890ce1d9917b1939b5d158b9ca1bc91dd/prog.net-main-lab5/prog.net-main-lab5/lab5/Client/Form1.cs#L10) — Створення конектора та підписка на події без втручання в їх внутрішню логіку.

### 2. Event-Driven Programming (Подійно-орієнтоване програмування)

Взаємодія між сервером та клієнтом побудована на подіях. Це дозволяє інтерфейсу залишатися "живим" (не зависати) під час очікування повідомлень.

- **Доказ у коді:** [Client/Form1.cs рядки 111-134](https://github.com/ipzk251gor-bot/KPZ/blob/228d7a1890ce1d9917b1939b5d158b9ca1bc91dd/prog.net-main-lab5/prog.net-main-lab5/lab5/Client/Form1.cs#L111) — Метод `OnMessage` спрацьовує автоматично при надходженні даних.

### 3. Принцип KISS (Keep It Simple, Stupid)

Метод оновлення стану кнопок та полів (`UpdateUi`) винесений в окрему функцію. Це робить код простим для читання та підтримки: один метод відповідає за один стан екрана.

- **Доказ у коді:** [Client/Form1.cs рядки 23-34](https://github.com/ipzk251gor-bot/KPZ/blob/228d7a1890ce1d9917b1939b5d158b9ca1bc91dd/prog.net-main-lab5/prog.net-main-lab5/lab5/Client/Form1.cs#L23) — Чітке розділення логіки доступності елементів керування.

### 4. Safe UI Threading (Безпечна робота з потоками)

Оскільки події від сервера приходять у фонових потоках, для оновлення елементів Windows Forms використовується `BeginInvoke`. Це запобігає критичним помилкам доступу до UI з іншого потоку.

- **Доказ у коді:** [Client/Form1.cs рядки 113-133](https://github.com/ipzk251gor-bot/KPZ/blob/228d7a1890ce1d9917b1939b5d158b9ca1bc91dd/prog.net-main-lab5/prog.net-main-lab5/lab5/Client/Form1.cs#L113) — Використання анонімних методів для безпечного оновлення списку чату та користувачів.
